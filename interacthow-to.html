What I Made and How it Works

I recreated my generative art this time incorporating the DOM. It takes the users input from the dropdown menu on the side and, based off of what the user selected, allows them to click on the art and have the item appear! From butterflies to ladybugs to flowers to dragonflies, this interactive art allows for the user to create a nature scene of their choice
Try clicking on the generative art above! (but, of course, don't forget to select something first)

".getElementById()" is a way for us to access all of the information under a certain id in the HTML file and bring it to our Javascript. I used ".getElementById()" to get things like the options inside the dropdown menu and the words inside the "--select--". Choices Made and Why

For our interactive webpage, we had the choice of making whatever we wanted as long as we met the following requirments: .getElementById, user input, a resulting output, and a conditional or function. Here is an overview of what I did for each requirement and why:

.getElementById

I used .getElementById to access aspects of the HTML and put it into the javascript. Using it didn't allow too much freedom nor creativity because it was a must in order to make an interactive webpage. In addition to getting each label/option under the dropdown menu, I also used .getElementById to create the subsections and the main section of the dropdown menu. I put these at the top to keep the code neat and to avoid having to use getElementById over and over again in random places in the code Here is an image of how I used .getElementById:

User Input

The user input I used in my final product was all in the dropdown menu. The user would "input" or select the bug/flower of their choice, and, once it was selected the user would be able to click onto the generative art and have it appear. I chose to make the dropdown menu as the user input to make sure that I was making something that any user could easily figure out what they were supposed to do without instruction. Something to Remember

One of my favorite parts of this class so far has been making art with programming. For this reason, I wanted to take this project as an opportunity to improve my old generative art. Although exciting, it presented many challenges. Going into this project, I had no idea how to incorporate a canvas that uses processing with javascript, CSS, and HTML. In order to continue using all of these elements together with art, here are some things I will make sure to remember.

Where do we ".getElementByID"? The canvas? The Javascript?

This was one of the most challenging parts of this project because, in order to start, I needed to know where to get the elements by their id. Over the course of this project I learned that, first, you cannot use ".getElementById" inside the canvas because the inside of the canvas is mainly processing. That being said, you can use ".getElementById" outside the canvas (above and below it). Because my project required two javascript files for the purposes of organization, I didn't use ".getElementById" on my canvas file, but it is definitely useful for the future.

How do we troubleshoot?

My interactive art required a lot of components and, therefore, opportunities for mistakes. When I was putting the different components together, a lot of the times it would freeze or the background wouldn't show up etc. This is where alert(); comes in. In some cases, when the dragonfly/flower/butterfly/ladybug wouldn't show up, I would put "alert();" at the top of the mouseClicked function and, if an alert showed, I would know that for sure the mouseClicked function was being called, and working. That narrowed down the problem being something inside the mouseClicked function that was either called in it or drawn in it. A limitation to this, however, is that this mainly works for only javascript and processing as far as I know, and I personally do not know a way of testing why an image isn't loading in HTML using "alert();".
